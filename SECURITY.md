# Security Policy

## Supported content

This repository contains documentation and public media references, not a runtime package. Security reports should concern malicious links, compromised media, credential exposure, unsafe example commands, or supply-chain risks in repository automation

## Reporting

Do not open a public issue for a vulnerability that could expose users. Use GitHub's private vulnerability reporting for this repository when available, or contact the EvoLink security team through the official channel listed on the EvoLink website

Never include API keys, access tokens, private prompts, or personal data in a report
